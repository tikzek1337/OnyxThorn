# Протокол синхронизации OnyxThorn

## Главные принципы

1. **End-to-end encryption**: сервер хранит только зашифрованные blob'ы.
   Все данные шифруются в браузере ключом, выведенным из пароля.
2. **User-initiated**: ничего не синхронизируется автоматически. Запрос
   делается только из `about:preferences#sync` через кнопки
   «Синхронизировать сейчас» / «Сделать резервную копию».
3. **Self-hosted**: один сервер = один пользователь (или семья).
4. **Минимум полей в open viewable form**: на сервере открыто хранится
   email, ник, URL аватара. Всё остальное — ciphertext.

## Криптография

| Параметр    | Значение                                  |
| ----------- | ----------------------------------------- |
| KDF         | PBKDF2-SHA256, 250 000 итераций           |
| Сольный размер | 16 байт (генерируется сервером при регистрации) |
| Симметричный шифр | AES-256-GCM                       |
| IV          | 12 байт случайных на каждое шифрование    |
| Tag         | 16 байт, авто-добавляется GCM             |

Пароль никогда не уходит с устройства в открытом виде — на сервер
отправляется только argon2id-хеш пароля (для входа). Это два разных
секрета:

* Пароль → **argon2id** → отправляется при логине (для аутентификации).
* Пароль + соль (получена после успешного логина) → **PBKDF2** → AES-ключ (для шифрования payload).

Соль уникальна для каждого аккаунта и **не меняется** при логинах.
Сервер возвращает её в ответ на `/auth/login` и `/auth/register`.

## Формат payload'ов

```jsonc
PUT /sync/bookmarks
{
  "iv":         "base64-encoded 12-byte IV",
  "ciphertext": "base64-encoded AES-GCM output (ciphertext + tag)",
  "meta": {
    "count":      42,             // не-чувствительная метадата для UI
    "client":     "OnyxThorn/0.1.0",
    "uploaded":   "2026-05-19T10:00:00Z"
  }
}
```

Сервер хранит то же самое 1-к-1 и отдаёт обратно по `GET`.

## Что синхронизируется

| Collection    | Что внутри (после расшифровки)                  |
| ------------- | ----------------------------------------------- |
| `bookmarks`   | Полное дерево закладок (`PlacesUtils.bookmarks.fetchTree()`) |
| `history`     | Последние 10 000 посещений с timestamp          |
| `passwords`   | Все `nsILoginInfo` из login-manager             |
| `settings`    | Подмножество prefs (см. `OnyxThornSync._dumpSettings`) |
| `tabs`        | Открытые вкладки во всех окнах                  |
| `addons`      | Установленные расширения + их настройки         |

## Полный workflow

```
[ client ]                                       [ server ]
   │
   │  POST /auth/register {email, password}
   ├────────────────────────────────────────────►│  argon2id hash
   │                                              │  generate salt
   │  {account, token, kdf_salt}                  │
   │◄────────────────────────────────────────────┤
   │
   │  derive AES key from (password, kdf_salt)
   │
   │  PUT /sync/bookmarks {iv, ciphertext, meta}
   ├────────────────────────────────────────────►│  store as-is
   │                                              │
   │  GET /sync/bookmarks                         │
   ├────────────────────────────────────────────►│
   │  {iv, ciphertext, meta, updated_at}          │
   │◄────────────────────────────────────────────┤
   │
   │  POST /backup {iv, ciphertext}              │  store + trim history to 30
   ├────────────────────────────────────────────►│
```

## Conflict resolution

Очень простое: **last-writer-wins**, на уровне всей коллекции. Никаких
diff/merge. Если коллекция «История» залилась с устройства A, а потом
залилась с устройства B — версия от B полностью перезапишет версию от A.

Перед заливкой клиент **сам** должен сделать merge:

1. `GET /sync/history` → расшифровать → текущая remote-версия.
2. Сделать union с локальными данными.
3. `PUT /sync/history` с объединённой версией.

## Backup'ы

* Хранятся в **отдельной таблице** `backups`, не пересекаются с
  `collections`.
* Содержат **всё**: bookmarks + history + passwords + settings + tabs + addons в одном JSON.
* Сохраняется **до 30** последних снапшотов на пользователя — старше
  удаляются автоматически.

## Что НЕ синхронизируется (нигде, никогда)

* Открытые приватные окна.
* Кэш / Cookie-jar (это локально — поведение «инкогнито»).
* Master-пароль (он же ключ деривации).
* `nss db` (sqlite-БД сертификатов) — у каждого устройства своя.

## Версионирование протокола

`GET /version` → `{ "protocol": 1, ... }`

Клиент при инициализации проверяет `protocol === 1` (или совместимый).
Если протокол не совпадает — синхронизация отключается, в UI показывается
«Сервер несовместим — обновите OnyxThorn».
